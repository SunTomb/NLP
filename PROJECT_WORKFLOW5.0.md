### 3.2 实验室集群交互式 Demo 部署指南

由于 Demo 系统使用 vLLM 进行高速推理，需要较大的 GPU 显存。在实验室集群（如 `Tang` 系列服务器）上运行时，需要配合 SSH 端口转发才能在本地浏览器中查看。以下是完整的操作指南：

**Step 1: 在服务器端启动 Demo**
登录到带有可用 GPU 资源的服务器（例如 `Tang-3-Wu` 或 `Tang-2-Wu`），进入项目环境并运行应用：

```bash
# 1. 登录服务器
ssh yesh@<集群IP>

# 2. 激活项目依赖环境
source /NAS/yesh/NLP/activate.sh

# 3. 进入 demo 目录并启动 (默认加载 outputs/generator_llama2_7b)
CUDA_VISIBLE_DEVICES=7 python demo/app.py
```

*注：程序默认绑定 `0.0.0.0:7860`。vLLM 启动过程可能需要几分钟加载权重，显存占用约 30-40GB，请确保运行前 GPU 显存充足。*

**Step 2: 建立本地 SSH 隧道映射**
由于集群节点的 `7860` 端口默认不对外暴露，您需要在**自己的本地电脑**上打开一个新的终端窗口，执行端口转发：

```bash
ssh -N -f -L 7860:127.0.0.1:7860 yesh@<集群IP>
```

*原理说明：该命令会在后台静默运行 (`-f -N`)，将本地机器的 `7860` 端口与服务器的 `7860` 端口连通。*

**Step 3: 浏览器直接访问**
在您的本地个人电脑上，打开 Chrome 或 Edge 浏览器并访问：

```text
http://localhost:7860
```

此时您将看到我们刚刚全新设计的 Self-RAG WebUI！您可以直接输入问题（例如 *"Who is the author of Harry Potter?"*），实时观察模型如何输出五彩斑斓的 `[Retrieval]`、`[Relevant]` 以及 `[Utility]` 等反思标签。
