## 一、研究问题与论文核心

### 1.1 Self-RAG 解决什么问题

传统 LLM 和 RAG 系统存在三个核心问题：

| 问题 | 传统 LLM 做法 | Self-RAG 的解决方案 |
|------|--------------|-------------------|
| **何时检索** | 不检索（纯参数化）或始终检索（标准 RAG） | 模型自主判断是否需要检索（`[Retrieve]` token） |
| **检索质量评估** | 盲目使用检索结果 | 模型评判检索内容的相关性（`[IsRel]` token） |
| **生成质量自检** | 无自我批判能力 | 模型评估生成内容是否被证据支持（`[IsSup]` token）以及整体有用性（`[IsUse]` token） |

### 1.2 Self-RAG 架构概览

```
用户输入 Query
      │
      ▼
Generator 开始生成
      │
      ├─ 输出 [No Retrieval] → 直接生成回答（无需外部知识）
      │
      └─ 输出 [Retrieval] → 触发检索
              │
              ▼
        Retriever 检索 Top-K 段落
              │
              ▼
        对每个段落，Generator 判断：
              ├── [IsRel]: Relevant / Irrelevant（段落是否相关）
              ├── 基于段落生成回答片段
              ├── [IsSup]: Fully / Partially / No Support（回答是否被证据支持）
              └── [IsUse]: Utility 1-5（回答总体有用性评分）
              │
              ▼
        根据反思 Token 的评分选择最佳生成结果
              │
              ▼
        最终回答
```

### 1.3 训练流程四步骤

Self-RAG 训练包含两个模型：**Critic**（教模型何时检索、如何评估）和 **Generator**（执行检索增强生成）。

| 步骤 | 内容 | 说明 |
|------|------|------|
| **Step 1** | Critic 训练数据生成 | 使用 GPT-4 为反思 token 生成标注数据 |
| **Step 2** | Critic 模型训练 | 在 Llama 2 基座上微调，学习预测 4 种反思 token |
| **Step 3** | Generator 训练数据生成 | 用训练好的 Critic + Retriever 为原始语料标注反思 token |
| **Step 4** | Generator 模型训练 | 在带反思标注的数据上微调 Llama 2，学习端到端的自适应检索与生成 |

### 1.4 评测任务覆盖

| 任务类型 | 数据集 | 评测指标 |
|----------|--------|----------|
| 短文本事实核查 | PubHealth | Accuracy |
| 科学推理 | ARC-Challenge | Accuracy |
| 短文本开放域 QA | TriviaQA, PopQA | EM / F1 |
| 长文本 QA | ASQA | Correctness, MAUVE, Citation Precision/Recall |
| 传记生成事实性 | FactScore | FactScore |

### 1.5 关键对比基线

- **Vanilla Llama 2**（无检索）
- **Standard RAG**（始终检索 + 拼接生成）
- **ChatGPT / Llama 2-Chat**
- **Retrieval-augmented Llama 2-Chat**
- **Perplexity.ai**（商用系统基线）

## 二、改进方案设计（冲高分核心）

> [!IMPORTANT]
> 课程明确写了"有开源代码的论文，如果在报告中体现复现结果和**进行改进优化**，会有适当加分"。以下改进方案是我们得高分的关键差异化。

### 2.1 改进方案总览

| ID | 改进方向 | 难度 | 展示效果 | 优先级 |
|----|---------|:----:|:--------:|:-----:|
| **I1** | **基座模型升级：Llama 2 → Llama 3 / Qwen 2.5** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 🔴 P0 |
| **I2** | **中文数据集适配与评测** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 🔴 P0 |
| **I3** | **新增反思维度：[IsFresh] 时效性 Token** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 🟡 P1 |
| **I4** | **Adaptive Top-K 动态检索策略** | ⭐⭐⭐ | ⭐⭐⭐⭐ | 🟡 P1 |
| **I5** | **交互式 Demo 系统构建** | ⭐⭐ | ⭐⭐⭐⭐⭐ | 🟡 P1 |
| **I6** | **与 DPO 训练范式对比** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 🟢 P2 |

### 2.2 各改进方案详解

#### I1: 基座模型升级（P0，必做）

**动机**：原论文使用 Llama 2 7B/13B，Llama 3 8B / Qwen 2.5 7B 在基础能力上有显著提升。

**方案**：
- 使用 `meta-llama/Llama-3.1-8B` 或 `Qwen/Qwen2.5-7B` 替换 Llama 2 7B
- 使用 `meta-llama/Llama-3.1-8B-Instruct` 替换 Llama 2 Chat 基线
- 对比升级前后在所有评测任务上的性能变化

**预期结论**：Self-RAG 框架具有**模型无关性**，在更强基座上仍能带来一致的提升。

#### I2: 中文数据集适配（P0，必做）

**动机**：原论文仅在英文数据上评测，中文适配体现本土化价值。

**方案**：
- 选择中文 QA 数据集：WebQA（百度）、DuReader（百度阅读理解）、CMMLU（中文多任务）
- 构建中文检索知识库（中文维基百科段落）
- 对比 Self-RAG vs 标准 RAG 在中文场景下的表现

#### I3: 时效性反思 Token [IsFresh]（P1）

**动机**：Self-RAG 没有考虑检索内容的时效性，过时的信息可能导致错误。

**方案**：
- 新增第 5 个反思 token `[IsFresh]`：`Fresh` / `Outdated` / `Timeless`
- 在 Critic 训练数据中加入时效性标注
- 在有时间敏感性的 QA 任务上验证效果

#### I4: Adaptive Top-K 动态检索（P1）

**动机**：原论文固定 Top-5/10 检索，不同查询的难度差异很大。

**方案**：
- 根据 `[IsRel]` token 的分布动态调整 K 值
- 简单问题检索少量文档（K=1-3），复杂问题检索更多（K=10-20）
- 在推理效率和准确性之间取得更好的平衡

#### I5: 交互式 Demo 系统（P1）

**动机**：一个可视化的 Demo 系统在报告展示中极具冲击力。

**方案**：
- 使用 Gradio/Streamlit 构建 Web UI
- 实时展示：输入问题 → 是否触发检索 → 检索到的段落 → 各反思 Token 的预测 → 最终回答
- 支持中/英文切换

#### I6: DPO 训练范式对比（P2，可选）

**动机**：原论文的 Generator 训练用标准 NTP 损失，DPO 可能带来进一步提升。

**方案**：
- 使用反思 Token 的评分构建偏好对：高 `[IsUse]` + `[Fully Supported]` 的输出为 chosen，低分输出为 rejected
- 在 Generator 训练阶段额外用 DPO 对齐
- 对比 NTP-only vs NTP+DPO 的性能差异

## 六、风险与应对

| 风险 | 概率 | 影响 | 应对方案 |
|------|:----:|:----:|---------|
| Critic 训练数据下载失败 | 低 | 🔴 高 | 使用 HuggingFace 镜像站下载；极端情况下用本地 LLM 替代 GPT-4 标注 |
| Generator 训练不收敛 | 低 | 🔴 高 | 检查 DeepSpeed 配置、学习率；直接使用官方预训练模型作为 fallback |
| Llama 3 适配失败 | 中 | 🟡 中 | Llama 3 与 Llama 2 架构一致，主要风险是 tokenizer 差异。确保在词表中正确注册反思 Token |
| 中文检索质量差 | 中 | 🟡 中 | 使用 BGE 系列中文 embedding 模型替代 Contriever |
| 复现结果与论文差距大 | 中 | 🟡 中 | 如实记录差异并分析原因（随机种子、数据版本等），差异分析本身也是加分内容 |
| 时间不够完成所有改进 | 中 | 🟡 中 | 保 I1（基座升级）和 I2（中文适配）为底线，I3-I6 按优先级取舍 |
| 该论文被其他组抢选 | 低 | 🔴 高 | 尽快联系助教确认选题；备选方案为 8.4 RLHF for Summarization |
